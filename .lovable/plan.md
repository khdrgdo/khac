
## نظرة عامة

موقع اجتماعي/أكاديمي لكلية تحتوي على 3 تخصصات (IT، نظم المعلومات، هندسة البرمجيات) مع 4 سنوات دراسية. الطلاب يسجلون بالرقم الجامعي، ينشرون ويتفاعلون ويتراسلون، والأساتذة يُضافون بالإيميل، والمشرفون يديرون الكورسات والمحتوى.

## الميزات الرئيسية

### 1) التسجيل والدخول
- التسجيل بـ **الرقم الجامعي** (فريد لا يتكرر) + الاسم + التخصص + السنة
- كلمة السر الأولى = نفس الرقم الجامعي (يُطلب تغييرها عند أول دخول)
- الأساتذة يدخلون بالإيميل (يُضيفهم المشرف)
- المشرف Admin له لوحة تحكم

### 2) الملف الشخصي
- الاسم، الرقم الجامعي، التخصص، السنة، صورة، نبذة، الدور (طالب/أستاذ/مشرف)

### 3) الكورسات
- منظمة حسب: التخصص ← السنة ← الفصل
- كل كورس: اسم، وصف، رابط (YouTube/Drive/PDF)، أستاذ، ملفات مرفقة
- الطلاب يتصفحون، المشرفون والأساتذة يضيفون/يعدلون

### 4) المنشورات (Feed)
- أي مستخدم مسجل ينشر (نص + صور اختيارية)
- تفاعلات متعددة: 👍 ❤️ 😂 😮 😢
- تعليقات مع ردود متداخلة
- حفظ ومشاركة المنشور
- تصفية: كل المنشورات / تخصصي / محفوظاتي

### 5) المراسلة
- محادثات خاصة 1-إلى-1
- مجموعات (يمكن إنشاؤها لكورس أو دفعة)
- رسائل نصية فورية

### 6) لوحة المشرف
- إدارة المستخدمين (تفعيل/حظر/تعيين دور)
- إضافة حسابات أساتذة بالإيميل
- إدارة التخصصات والكورسات
- حذف منشورات/تعليقات مخالفة

## هيكل الصفحات

```
/                     صفحة رئيسية (feed) — تتطلب دخول
/auth                 تسجيل / دخول
/change-password      تغيير كلمة السر لأول مرة
/courses              كل الكورسات (فلترة بالتخصص/السنة)
/courses/:id          تفاصيل كورس
/feed                 المنشورات
/posts/:id            منشور مع تعليقاته
/messages             قائمة المحادثات
/messages/:id         محادثة مفتوحة
/profile/:id          ملف مستخدم
/saved                المحفوظات
/admin                لوحة المشرف
```

## التصميم

سأقترح 3 اتجاهات تصميم (بعد الموافقة) — أكاديمي عصري، عربي RTL بالكامل، مع اهتمام بالوضوح والقراءة.

## التفاصيل التقنية

**Backend:** Lovable Cloud (قاعدة بيانات + مصادقة + تخزين ملفات + Realtime للرسائل)

**جداول قاعدة البيانات:**
- `profiles` — user_id, university_number (unique), full_name, major, year, avatar, bio, must_change_password
- `user_roles` — user_id, role (student/teacher/admin) — جدول منفصل للأمان
- `majors` — id, name (IT/IS/SE)
- `courses` — id, major_id, year, semester, name, description, teacher_id
- `course_links` — id, course_id, title, url, type
- `posts` — id, author_id, content, images
- `post_reactions` — post_id, user_id, type (like/love/haha/wow/sad)
- `comments` — id, post_id, parent_id (للردود), author_id, content
- `saved_posts` — user_id, post_id
- `conversations` — id, is_group, name
- `conversation_members` — conversation_id, user_id
- `messages` — id, conversation_id, sender_id, content, created_at

**الأمان:** RLS على كل الجداول، دور `admin` عبر `has_role()` security definer، سياسات صارمة (كل مستخدم يعدل/يحذف محتواه فقط، الرسائل تُقرأ فقط من أعضاء المحادثة).

**التسجيل بالرقم الجامعي:** نحوّله داخليًا إلى إيميل وهمي مثل `{رقم}@college.local` لأن Supabase يتطلب إيميل، والرقم الجامعي يُحفظ في `profiles` مع قيد UNIQUE. Trigger على `auth.users` ينشئ profile تلقائيًا.

**Realtime:** للرسائل والتعليقات الجديدة.

## خطة التنفيذ على مراحل

1. **الأساس:** تفعيل Lovable Cloud، إنشاء الجداول والسياسات، صفحة التسجيل/الدخول بالرقم الجامعي، تغيير كلمة السر لأول مرة
2. **الملفات الشخصية والصفحة الرئيسية:** Layout، Header بالتنقل، صفحة profile
3. **الكورسات:** عرض، فلترة، إضافة (للأساتذة/المشرف)
4. **المنشورات:** إنشاء، عرض، تفاعلات، تعليقات وردود، حفظ
5. **المراسلة:** محادثات 1-1 ومجموعات مع Realtime
6. **لوحة المشرف:** إدارة المستخدمين والأساتذة والكورسات

- شعار الكلية واسمها الرسمي
